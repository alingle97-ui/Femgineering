Your First Flask App

Lesson 1 · Femgineering Dashboard

Right now the dashboard is just a PRD. This lesson gets you a real, running web page — served by code you wrote — as the first brick of the actual app.

What Flask is

A "web app" is just a program that, instead of printing to your terminal, sends back a page when a browser asks for one. Flask is a small Python library that handles the browser-talking part, so you can write a plain Python function and say "run this when someone visits /."

You already checked out Python 3.9 and pip (Python's package installer) — both already on your machine, nothing to install there.

Step 1 — Install Flask

In your terminal, in the Femgineering folder, run:

python3 -m pip install flask

This downloads Flask and makes it available to import in your Python code.

Step 2 — Write the app

Create a new file called app.py in the Femgineering folder with exactly this:

from flask import Flask

app = Flask(__name__)

@app.route("/")
def home():
    return "<h1>Femgineering Dashboard</h1><p>It's alive.</p>"

if __name__ == "__main__":
    app.run(debug=True)

Line by line, since every piece here will show up again:

  • Flask(__name__) creates your app — think of it as "the thing that will answer browser requests."
  • @app.route("/") is a decorator — it tells Flask "when someone visits the root address (/), run the function right below me."
  • def home(): is that function. Whatever it returns is what the browser displays.
  • debug=True means Flask restarts itself automatically when you save changes to the file, and shows you readable errors in the browser instead of a blank page.

Step 3 — Run it

python3 app.py

You should see output mentioning something like Running on http://127.0.0.1:5000. Open that address in your browser.

Win: a page reading "Femgineering Dashboard — It's alive." — rendered by a server you started, from code you wrote.

Leave the terminal running while the app is up. Ctrl+C in the terminal stops it.

Try it yourself

Before moving on, without looking back at Step 2:

  1. Add a second route, /about, that returns a short sentence about the podcast.
  2. Visit http://127.0.0.1:5000/about to confirm it works. (Debug mode means you don't need to restart the server after saving.)

This is deliberately unaided — figuring it out from the pattern above is what makes it stick, not just watching it work.
